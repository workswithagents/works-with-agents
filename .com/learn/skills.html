Foundation

Pattern 2: Skills

Reusable procedural knowledge. Build once, load on demand. Never re-explain.

⏱ 18 min read 🛠 45 min for first skill 📋 Prerequisites: Pattern 1 (Boot — your agent has an AGENTS.md)

What Most People Do Wrong

You tell your agent how to do something. It does it. Next session, you tell it again. Every single time. You become the skill library — your brain holds all the procedural knowledge. The agent forgets everything between sessions.

The Lesson

The Skill Template

Every skill follows this format. Save as SKILL.md in your skills directory:

---
name: skill-name-lowercase-hyphens
description: One sentence — what this skill does
triggers:
  - "keyword that signals this task"
  - "another keyword"
---

# Skill Name

## Trigger
Use when [specific situation].

## Steps
1. **Step name**
   ```bash
   exact command
   ```

## Pitfalls
- Common mistake and how to avoid it

## Verification
- How to confirm it worked

When to Create a Skill

  • You've done something 3+ times in separate sessions
  • You found a tricky edge case and don't want to re-discover it
  • A task required multiple clarification rounds — capture the final answer
  • You discovered a workflow that worked better than expected
  • After any complex task (5+ tool calls) that succeeded

When to Load vs. Skip

Not every skill should load every session. The trigger keyword system is the gate: skills load only when the conversation matches their triggers. Design your triggers to be specific enough that they don't fire accidentally, but broad enough that they catch all relevant tasks.

Skill Library Growth

The compounding effect: session 1 you have 0 skills. Session 10 you have 3. Session 50 you have 15. Session 100 you have 40. Each skill saves 5-15 minutes of re-explaining. At 40 skills × 10 minutes saved × weekly usage, that's hours of recovered time every month — and the agent handles increasingly complex tasks because it already knows the fundamentals.

Vocabulary

Skill
A markdown document containing step-by-step instructions, pitfalls, and verification steps — loaded by the agent when a task matches its trigger keywords.
Skill library
A directory of reusable skills shared across agents and projects — grows with every successful task completion.
Trigger keyword
Words or phrases that cause the agent to auto-load a skill (e.g. 'deploy' loads the deploy skill, 'fix SPFx build' loads spfx-heft-build-breakfix).
Pitfall
A documented failure mode — bugs found by one agent, learned by all. Stored in the Pitfall Registry.
Skill authoring
The craft of writing a skill that another agent (or human) can follow without assumptions. Includes: trigger, steps, pitfalls, and verification.
