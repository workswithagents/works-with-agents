Pattern 10: Compounding

The feedback loop. Discoveries become skills. Skills compound. Agents that get better every session.

⏱ 12 min read 🛠 Ongoing — every session adds something 📋 Prerequisites: All previous patterns

What Most People Do Wrong

They treat every session as independent. The agent solves today's problem but learns nothing. Tomorrow, the same problem takes just as long. The agent is a calculator, not a colleague — it computes but never improves.

The Lesson

The Compounding Loop

Session → Task completed → Discovery captured → Skill saved
   ↓                                                    ↓
Agent is faster next time ← Skill loaded automatically ← Next session

What to Capture

  • Successful approaches: "The way we fixed that SPFx build error — save it as a skill." Now the agent can fix it in 30 seconds next time.
  • Pitfalls discovered: "Python 3.12 breaks that library." Save to Pitfall Registry so other agents never hit it.
  • Environment quirks: "The cloudflare API token is stored in credential-proxy under 'cloudflare API' not 'Cloudflare API Token'." Save as a memory fact.
  • Tool optimisations: "Use rsync not scp for large directories." Save as a skill step.

The Compounding Maths

Session 1: Agent knows nothing. Every task is slow. Session 10: Agent knows your project structure, Python version, testing conventions. 30% faster. Session 50: Agent has 15 skills, knows all the pitfalls, anticipates your preferences. 2× faster. Session 100: Agent has 40 skills, shared pitfall registry, compound memory. 3-5× faster than session 1.

The bend in the curve happens around session 15-20. That's when enough skills and memory have accumulated that the agent starts teaching you new patterns rather than you teaching it. Most people quit at session 5. The ones who push through to session 30 never go back.

Vocabulary

Compounding loop
The cycle: agent solves a problem → captures the solution as a skill → future sessions are faster → agent tackles harder problems → discovers new solutions.
Discovery
A new technique, pitfall, or workflow uncovered during a session — captured before it's forgotten. Every session produces at least one discovery.
Skill library growth
The compounding metric: skills written this month, total skills available, skills loaded per session. The rate of growth IS the compounding rate.
Compounding maths
1% better each session over 100 sessions = 2.7× improvement. The curve is exponential, not linear. Most people quit before they see the bend.
