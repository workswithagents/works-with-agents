Autonomy

Pattern 4: Decision Protocols

Autonomy without chaos. Rules your agent follows. When to ask, when to decide.

⏱ 12 min read 🛠 15 min to write protocols 📋 Prerequisites: Patterns 1-3 (Foundation complete)

What Most People Do Wrong

The agent asks permission for everything. 'Can I create a file?' 'Can I run this test?' 'Can I search for that pattern?' You become the bottleneck — spending more time approving than the agent saves. Or the opposite: you give it full autonomy and it deploys untested code to production.

The Lesson

The Autonomy Spectrum

Not all actions are equal. Some should always require approval. Some should never. Most fall in between:

  • Always auto: Read files, search code, run non-destructive commands, check git status, lint
  • Auto if tested: Write files (post-syntax-check), run tests, commit (post-lint+test), create branches
  • Ask first: Deploy to production, delete branches, modify CI config, send external messages, modify credentials
  • Never auto: Delete data, drop databases, revoke access, change billing, modify production config

The Protocol Template

# Decision Protocols

## Auto-proceed (no approval needed)
- Read files, search code, run non-destructive commands
- Create new files (syntax check runs automatically)
- Targeted edits with patch tool
- Run tests, lint, type-check

## Ask before
- Deploy to production
- Modify CI/CD configuration
- Delete branches or files
- Send external communications

## Never auto (always require explicit confirmation)
- Delete data or databases
- Modify production credentials
- Change billing or access controls

Trust Calibration

Start at Level 1 (non-destructive auto only). After 10 sessions with no incidents, graduate to Level 2 (auto with post-hoc review). After 50 sessions and zero rollbacks, consider Level 3. Trust is earned per category — an agent might be trusted on code changes but not on deployment. Calibrate separately.

Vocabulary

Decision protocol
A rule that tells the agent when to proceed autonomously and when to pause for human approval.
Micromanagement trap
The pattern of over-approving — the human becomes a bottleneck because the agent asks permission for every file write, every command, every decision.
Trust calibration
The process of starting with tight constraints (always ask) and loosening over time as the agent proves reliable on specific categories of action.
Destructive guard
A protocol that requires explicit approval for irreversible actions: deploy to production, delete data, send external email, publish to social media.
Autonomy level
The tiered permission system: Level 0 (ask everything) → Level 1 (non-destructive auto) → Level 2 (auto with post-hoc review) → Level 3 (full auto, monitor-only).
