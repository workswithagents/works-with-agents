Pattern 5: Tool Composition

The right instrument for each task. The difference between 30 seconds and 15 minutes.

⏱ 15 min read 🛠 Immediate — decisions every session 📋 Prerequisites: Patterns 1-4

What Most People Do Wrong

They delegate everything to sub-agents. Every code change becomes a delegate_task. Result: 600-second timeouts, garbage output, and a context window full of sub-agent summaries. Or they do the opposite: run everything through terminal. cat to read files, sed to edit. Ten shell commands where one write_file would do.

The Lesson

The Tool Decision Matrix

Task              | Right tool      | Wrong tool       | Cost of wrong
Read a file       | read_file       | cat in terminal  | No line numbers, no pagination
Search code       | search_files    | grep -r          | Slower, no fuzzy matching
Edit a file       | patch           | sed or rewrite   | Regex failures, overwrites
Create a file     | write_file      | echo >>          | Escaping bugs, no syntax check
Run a command     | terminal        | N/A              | This IS the right tool
Long computation | execute_code    | Multiple terminals| Context pollution
Research          | web_search      | terminal curl    | Manual browsing, no synthesis
Parallel work     | delegate_task   | Sequential       | 3x slower
Complex reasoning | Direct analysis | delegate_task    | Sub-agent has no context

The Golden Rule

Never delegate coding to a sub-agent.

Sub-agents have no memory of your conversation. They don't know your project structure, conventions, or corrections. They generate entire files instead of targeted edits. The result: overwritten working code, invented directories, and output that needs complete rewrites.

For coding: use write_file, patch, and terminal directly. Sub-agents are for research, parallel analysis, and reasoning-heavy tasks — not code generation.

When to Use delegate_task

  • Yes: Research topic A and B in parallel, code review, deep analysis of a separate codebase, synthesizing information from multiple sources
  • No: Writing code, editing files, running tests, anything that needs your project context, any single-step operation

Vocabulary

Tool set
The collection of capabilities an agent has — terminal, file operations, web search, code execution, delegation. Each has a specific purpose.
Tool misuse
Using the wrong tool for a task: terminal sed instead of patch, cat instead of read_file, delegate_task for a single file write.
Delegation
Spawning sub-agents for complex subtasks. Good for parallel research and code review. Bad for code generation (sub-agents have no context).
Tool budget
The total number of tool calls per turn — each call costs tokens and time. Composing tools efficiently saves both.
