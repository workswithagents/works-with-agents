Pattern 7: Pipelines

Agents that run while you sleep. Scheduled, event-driven, unattended.

⏱ 15 min read 🛠 30 min for first pipeline 📋 Prerequisites: Patterns 1-6

What Most People Do Wrong

Agents are treated as interactive tools only — you must be present for them to work. Builds wait until morning. Monitoring requires checking dashboards. Research happens only when you remember to ask. The agent is a power tool you have to operate, not a worker you can schedule.

The Lesson

The Pipeline Pattern

  1. Define the trigger: Cron schedule (every morning at 8am), webhook (on push to main), or event (when disk usage exceeds 80%)
  2. Write a self-contained prompt: Everything the agent needs to complete the task — file paths, API endpoints, expected output format, delivery instructions. No user will be there to clarify.
  3. Set the delivery target: Where does the output go? Telegram message, Discord channel, email, local file, or pushed to another pipeline?
  4. Add a watchdog: A second agent that monitors the first. If the pipeline misses 3 consecutive runs, the watchdog fires an alert.

Common Pipeline Types

  • Daily briefing: Collect news, prices, metrics → summarise → deliver to Telegram at 8am
  • Build pipeline: On push → run tests → deploy if green → notify on Telegram
  • Research pipeline: Search for topic X daily → track changes → alert on significant findings
  • Health check: Poll endpoints every 5 minutes → silent if healthy → alert if degraded
  • Integration tracker: Check GitHub issues daily → detect new responses → flag opportunities

Pipeline Chains

Pipeline A (collector) runs at 7am, gathers raw data, writes to a file. Pipeline B (processor) runs at 7:30am, reads that file, analyses it, and delivers the summary. Use the context_from parameter on cron jobs to chain them: Agent B's prompt includes Agent A's most recent output automatically.

Vocabulary

Cron job
A scheduled agent invocation — runs at a fixed time or interval with a self-contained prompt. No user present, so the prompt must include everything the agent needs.
Pipeline composition
Chaining multiple agents together where the output of one feeds the input of the next. Agent A collects data, Agent B processes it.
Watchdog
A monitoring agent that checks system health and alerts only when something is wrong. Silent when healthy, loud when broken.
Webhook trigger
Event-driven agent invocation via HTTP POST — runs when an external system signals (GitHub push, Stripe event, CI completion).
Delivery target
Where pipeline output lands — Telegram, Discord, email, or local file. Each pipeline can route results to different destinations.
