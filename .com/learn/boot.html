Foundation

Pattern 1: Boot

First session setup. What must be in place before you type a single prompt.

⏱ 15 min read 🛠 30 min to apply 📋 Prerequisites: An AI coding agent installed (Claude Code, Codex, Hermes, or similar)

What Most People Do Wrong

They open a terminal, type a prompt, and start coding. The agent has no idea what project it's in, what Python version to use, or where anything lives. Every command needs qualification. Every tool call needs vetting. By the time the agent is oriented, you've burned half the session.

The Lesson

The 5-Field AGENTS.md Template

Save this as AGENTS.md in your project root. Fill in the bracketed fields:

# AGENTS.md — [YOUR PROJECT NAME]

[ONE SENTENCE: what this project is]

## Stack
- [LANGUAGE + EXACT VERSION] at [EXACT PATH]
- [ADD ANY OTHER CRITICAL TOOLS]

## Conventions
- Run tests with: [EXACT COMMAND]
- Lint with: [EXACT COMMAND]
- [GIT WORKFLOW RULE]
- [AUTONOMY RULE]

## Key files
- `[PATH TO MAIN ENTRY POINT]`
- `[PATH TO TESTS]`
- `[PATH TO CONFIG]`
- `[PATH TO DEPLOY SCRIPT]`

Why This Works

The AGENTS.md is the difference between a 15-minute setup and a 2-minute setup. Every time. The agent reads it once and knows: where the tests are, what Python version to use, whether to lint before committing, and when to ask permission. Without it, you're onboarding a new developer every session.

When your agent loads the AGENTS.md, it immediately knows the project's identity, stack, conventions, and key file locations. No questions asked. No corrections needed. The first tool call lands exactly where it should.

Beyond AGENTS.md — What Else to Seed

  • Memory seeding: Save durable facts like Python version, project structure, and your preferences before session 1. The agent should never have to ask "what version of Python do you use?"
  • Skill preloading: If your project always needs certain skills (testing, deployment, linting), load them in the boot sequence rather than waiting for the agent to discover them.
  • Decision rules: Set ground rules early: "Proceed without asking on non-destructive changes" or "Always lint before committing."

The Boot Checklist

  1. Create AGENTS.md with project identity, stack, conventions, and key files
  2. Seed initial memory with durable facts (Python version, paths, preferences)
  3. Preload critical skills your project always needs
  4. Set autonomy ground rules
  5. Verify: start a fresh session and count how many questions the agent asks. Target: zero.

Vocabulary

AGENTS.md
A project-level file that tells the agent about your codebase, conventions, and constraints — the agent's source of truth for every session.
System prompt
Instructions injected at the start of every conversation that define the agent's personality, boundaries, and tool capabilities.
Environment config
Persistent settings (model, tools, working directory) that survive across sessions so the agent doesn't ask 'what Python version?' every time.
Initial memory
Durable facts seeded before the first session — language versions, project structure, user preferences, coding conventions.
Boot script
A script or AGENTS.md file that runs once to initialise the agent with everything it needs to know before it starts working.
