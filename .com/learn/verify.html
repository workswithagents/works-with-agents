Pattern 9: Verify

Trust but verify. Automated gates catch problems — and there's a process for when they don't.

⏱ 13 min read 🛠 15 min to set up first gates 📋 Prerequisites: Patterns 1-8

What Most People Do Wrong

Agent modifies a file, saves it, commits it. The syntax error isn't caught until CI runs — 20 minutes later. Or a UI change breaks the layout and nobody notices for a week. Trust without verification is just hope.

The Lesson

The Quality Gate Stack

  1. Syntax gate (instant): AST parsing runs on every write_file and patch. Missing parenthesis? Blocked. Invalid import? Blocked. The file is never saved with broken syntax.
  2. Lint gate (instant): Runs after syntax check. Catches style violations, unused imports, naming inconsistencies. Only NEW errors are shown — pre-existing ones are filtered out.
  3. Type check (fast): Runs on Python files. Catches type mismatches before they become runtime errors.
  4. Test gate (slower): Runs the full test suite. Only triggered on significant changes, not every save.
  5. Visual QA (post-deploy): Automated screenshots of all pages after a deploy. The agent compares light and dark themes, checks for layout breaks.

Human Review Cadence

  • Daily: Review git diffs from the previous day's agent work. 5 minutes.
  • Weekly: Review architecture decisions, new files created, patterns established. 15 minutes.
  • Monthly: Audit the full project: are quality gates still firing? Are there false positives to tune? Is the skill library up to date? 30 minutes.

When to Intervene

Intervene when: a gate fires and the agent can't resolve it after 3 attempts, the same pattern of errors repeats across sessions, the agent is working around a gate instead of fixing the root cause, or a production issue slipped through all gates — that means the gates need strengthening.

Vocabulary

Quality gate
An automated check that runs after every agent action — syntax check, lint, type check, test suite. Blocks bad code before it's committed.
Syntax gate
AST parsing that catches broken code immediately — the agent can't save a file with a missing parenthesis or invalid import.
Visual QA
Automated screenshot capture for UI changes — the agent compares before/after and flags unexpected visual differences.
Review cadence
The rhythm of human review: what to check daily (diffs), weekly (architecture), monthly (strategy) — and when to intervene.
False positive
A gate that blocks valid work — the most common reason humans disable quality gates. Design gates to be strict on real errors, lenient on style.
