Phase 1 — Foundation Modules 1–3

Module 1

Boot

First session setup. What must be in place before you type a single prompt.

Vocabulary
AGENTS.md
A project-level file that tells the agent about your codebase, conventions, and constraints — the agent's source of truth.
System prompt
Instructions injected at the start of every conversation that define the agent's personality, boundaries, and capabilities.
Environment config
Persistent settings (model, tools, working directory) that survive across sessions.
Initial memory
Durable facts seeded before the first session — language versions, project structure, user preferences.
Resources
  • Boot skill reference — canonical setup checklist
  • Methodology overview — how the 10 patterns fit together
  • Knowledge Platform — agent-discoverable API docs
What You'll Learn
  • Writing an AGENTS.md that actually helps the agent
  • Configuring environment variables that persist across sessions
  • Seeding initial memory so the agent never asks "what version of Python?"
Module 2

Skills

Reusable procedural knowledge. Build once, load on demand.

Vocabulary
Skill
A markdown document containing step-by-step instructions, pitfalls, and verification steps — loaded by the agent when a task matches its triggers.
Skill library
A directory of reusable skills shared across agents and projects.
Trigger keyword
Words or phrases that cause the agent to auto-load a skill (e.g. "deploy" → deploy skill).
Skill authoring
The craft of writing a skill that another agent (or human) can follow without assumptions.
Resources
  • Skill Registry — browse published skills
  • Skill specification — authoring format
  • Example skills: spfx-local, systematic-debugging
What You'll Learn
  • Creating your first reusable skill
  • When to load a skill vs. when to skip
  • Organising a skill library for teams
Module 3

Memory

Durable context across sessions. Preferences, corrections, facts — remembered. Never re-explain.

Vocabulary
Session memory
Context that exists only within the current conversation — lost when the session ends.
Durable memory
Persistent storage that survives across sessions and agent restarts.
Fact
A single atomic piece of knowledge (entity-attribute-value) stored in the fact base.
Memory compaction
Periodic summarisation of long conversations to stay within context limits without losing important detail.
Honcho
A dialectic memory layer that observes conversations and builds structured peer profiles over time.
Resources
  • FactBase API — query and store facts
  • Memory architecture spec
  • Pattern: Memory — the agent that remembers
What You'll Learn
  • What to save vs. what to let go
  • Memory architecture: facts, preferences, corrections
  • Search and retrieval patterns across sessions

Phase 2 — Autonomy Modules 4–5

Module 4

Decision Protocols

Autonomy without chaos. Rules your agent follows. When to ask, when to decide.

Vocabulary
Decision protocol
A rule that tells the agent when to proceed autonomously and when to pause for human approval.
Micromanagement trap
The pattern of over-approving — the human becomes a bottleneck because the agent asks permission for everything.
Trust calibration
The process of starting with tight constraints (always ask) and loosening over time as the agent proves reliable.
Destructive guard
A protocol that requires explicit approval for irreversible actions (deploy, delete, email external).
Resources
  • Decision Protocols spec
  • Decision protocol pitfalls — common mistakes
  • Blueprint Registry — verified agent configs with built-in protocols
What You'll Learn
  • Writing effective decision rules that don't create bottlenecks
  • Trust calibration: start tight, loosen over time
  • When to ask vs. when to decide — the framework
Module 5

Tool Composition

The right instrument for each task. The difference between 30 seconds and 15 minutes.

Vocabulary
Tool set
The collection of capabilities an agent has — terminal, file operations, web search, code execution, delegation.
Tool misuse
Using the wrong tool for a task (e.g. terminal sed instead of patch, cat instead of read).
Delegation
Spawning sub-agents for complex subtasks — parallel execution, isolated context.
Tool budget
The total number of tool calls an agent can make per turn — composes with cost and latency constraints.
Resources
  • Tool Composition spec
  • Spike skill — throwaway experiments pattern
  • Agentic Delegation — multi-agent handoff protocol
What You'll Learn
  • When to use terminal vs. file operations vs. delegation
  • Avoiding tool misuse patterns that waste tokens and time
  • Building the right defaults for your tech stack

Phase 3 — Scale Modules 6–7

Module 6

Orchestration

Multi-agent workflows. Complex work split across specialist agents working in parallel.

Vocabulary
Orchestrator
A parent agent that decomposes tasks and spawns specialist sub-agents — coordinates results.
Specialist agent
A focused agent with a narrow role: researcher, builder, reviewer, documentor.
Parallel delegation
Running multiple sub-agents simultaneously on independent subtasks.
Handoff
The protocol for passing work and context between agents — what information travels with the task.
Agent mesh
A persistent network of agents that communicate via a shared state layer (SQLite+WAL, message bus).
Resources
  • Orchestration spec
  • Agent Orchestrator skill
  • Agent State DB — shared state for agent meshes
  • Bastion Gateway — fleet health monitoring
What You'll Learn
  • When one agent isn't enough — recognising orchestration signals
  • Role-based agent design (researcher, builder, reviewer)
  • Managing agent communication and handoffs
Module 7

Pipelines

Agents that run while you sleep. Scheduled, event-driven, unattended.

Vocabulary
Cron job
A scheduled agent invocation — runs at a fixed time or interval with a self-contained prompt.
Pipeline composition
Chaining multiple agents together where the output of one feeds the input of the next.
Watchdog
A monitoring agent that checks system health and alerts only when something is wrong.
Webhook trigger
Event-driven agent invocation via HTTP POST — runs when an external system signals.
Delivery target
Where pipeline output lands — Telegram, Discord, email, or local file.

Phase 4 — Harden Modules 8–10

Module 8

Resilience

Failure recovery that doesn't require human intervention. The agent finds another way.

Vocabulary
Exponential backoff
Retrying with increasing delays (1s, 2s, 4s, 8s…) — prevents hammering a failing service.
Circuit breaker
Stops retrying after N consecutive failures — prevents cascading failures in pipelines.
Transient error
A temporary failure that resolves on retry (network timeout, rate limit).
Permanent error
An error that won't resolve on retry (invalid API key, missing file) — should be escalated, not retried.
Self-healing
The agent detects its own failure, diagnoses the cause, and applies a fix without human input.
Resources
  • Resilience spec
  • Model Fallback — automatic provider failover
  • Cron Guard — detects stuck pipelines
  • Pitfall Registry — failures other agents have hit and how they recovered
What You'll Learn
  • Error categorisation: transient vs. permanent
  • Retry strategies with backoff and circuit breakers
  • Self-healing: when the agent fixes its own mistakes
Module 9

Verify

Trust but verify. Automated gates catch problems — and there's a process for when they don't.

Vocabulary
Quality gate
An automated check that runs after every agent action — syntax check, lint, type check, test suite.
Syntax gate
AST parsing that catches broken code before it's committed — blocks the agent from saving invalid files.
Visual QA
Automated screenshot capture for UI changes — the agent compares before/after and flags differences.
Review cadence
The rhythm of human review: what to check daily, weekly, monthly — and when to intervene.
False positive
A gate that blocks valid work — the most common cause of humans disabling quality gates.
Resources
  • Verify spec
  • Screenshot After Changes — visual QA automation
  • Pre-Commit Review — quality gate checklist
  • Project Audit — comprehensive gap analysis
What You'll Learn
  • Automated testing that runs after every agent action
  • Syntax, linting, type checking — the safety net
  • Human review cadence: what to look for, when to intervene
Module 10

Compounding

The feedback loop. Discoveries become skills. Skills compound. Agents that get better every session.

Vocabulary
Compounding loop
The cycle: agent solves a problem → captures the solution as a skill → future sessions are faster → agent tackles harder problems.
Discovery
A new technique, pitfall, or workflow uncovered during a session — captured before it's forgotten.
Skill library growth
The compounding metric — skills written this month, total skills available, skills loaded per session.
Compounding maths
1% better each session over 100 sessions = 2.7× improvement. The compounding curve is exponential, not linear.
Resources
  • Compounding spec
  • Agent Self-Improver — auto-captures learnings
  • Post-Task Capture — save discoveries as skills
  • FactBase — cumulative knowledge store
What You'll Learn
  • Saving successful approaches as reusable skills
  • Pattern recognition across sessions and projects
  • The compounding maths: 1% better each session, forever
